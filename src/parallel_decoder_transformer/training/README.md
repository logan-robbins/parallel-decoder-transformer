# Training Pipeline

Training stack for the Parallel Decoder Transformer (PDT) with a frozen GPT-OSS trunk and trainable sidecar coordination modules.

This document focuses on the current code paths, configs, and outputs generated by running the trainer.

See [PAPER.md](../../../docs/arxiv_submission/PAPER.md) Appendix A for the paper-level objectives.

## Prerequisites

1. Run the dataset pipeline and produce `kd_train.jsonl`, `kd_validation.jsonl`, and `kd_test.jsonl`
2. Place GPT-OSS-20B weights under `gpt-oss-20b/original/` or update `model.trunk.base_model`
3. Install dependencies with `uv sync`
4. Use CUDA GPU for practical training

```bash
uv sync
uv run python -c "import torch; print({'cuda': torch.cuda.is_available(), 'mps': hasattr(torch.backends, 'mps') and torch.backends.mps.is_available()})"
```

## Training Architecture

Every training step runs two branches through the same frozen-trunk PDT model:

1. Teacher branch: conditioned on teacher notes from the dataset pipeline
2. Student branch: conditioned on speculative or degraded notes and trained to match the teacher-side targets

`teacher.type` controls whether the teacher branch is:

- `stop_grad`: the same model with detached gradients
- `ema`: a separate EMA teacher model

Teacher notes are read from dataset artifacts. Training does not make live LLM API calls.

## 4-Stage Curriculum

| Stage | Name | Main trainable components | Role |
|------:|------|---------------------------|------|
| 0 | Planner pretrain | Planner head, notes head, `plan_notes_proj` | Learn latent plan slots and snapshot-0 contract |
| 1 | Stream bootstrap | Stream adapters, SNC cross-attention, stream classifier | Learn stream-specific conditioning under teacher notes |
| 2 | Bus enablement | Speculation head plus prior trainable modules | Learn to write latent summaries into the notes bus |
| 3 | Commit control | Agreement head, coverage head plus prior trainable modules | Learn readiness / ownership behavior for synchronized continuation |

## Loss Surface

| Loss | Code path | Purpose |
|------|-----------|---------|
| `L_plan` | planner CE | Latent planner-slot prediction |
| `L_notes` | notes MSE | Align notes head with teacher notes |
| `L_spec` | speculation MSE | Align speculation head with teacher notes |
| `L_LM-CE` | token CE | Language-model supervision |
| `L_LM-KD` | planner/LM KD | Distill teacher branch distributions |
| `L_cov` | coverage BCE | Plan-item ownership / coverage supervision |
| `L_ready` | agreement BCE | Continuation readiness supervision |

## Config

### `configs/canonical.yaml`

Paper-aligned canonical config for multi-GPU training.

- `attn_implementation: "flash_attention_2"` (HF pass-through)
- `gradient_checkpointing: true`
- `device_map: null` (DDP mode)
- `unfreeze_top_n_layers: 0` (fully frozen trunk)
- `max_steps: 50000`
- Stages 0–3 only

## Running Training

### Multi-GPU (canonical)

```bash
uv run torchrun --nproc_per_node=N scripts/train_wandb.py \
  --config configs/canonical.yaml
```

### Single-GPU

```bash
uv run scripts/train.py --config configs/canonical.yaml
```

## What the Trainer Writes

Artifacts are written under `training.telemetry_dir` (default `experiments/gpt_oss/`).

| File | Contents |
|------|----------|
| `adapters_step_{N}.pt` | Checkpoint with adapter weights and training state |
| `adapters.pt` | Final adapter-only state dict |
| `train_manifest.json` | Run metadata, config path, dataset paths, thresholds, git SHA |
| `train_run_stages.json` | Stage transition history |
| `training_report.json` | Aggregated metric summary |
| `agreement_thresholds.json` | Agreement-threshold ROC sweep and selected threshold |
| `coverage_thresholds.json` | Coverage-threshold sweep statistics |

## Threshold Calibration

- Agreement threshold: auto-calibrated during training from ROC statistics and written to `agreement_thresholds.json`
- Coverage threshold: evaluated across a sweep and written to `coverage_thresholds.json`, but not auto-applied back into config

## Key Code Paths

- [trainer.py](trainer.py): stage management, forward passes, losses, checkpointing, telemetry
- [dataset.py](dataset.py): lazy JSONL dataset loader
- [collator_kd.py](../../data/collator_kd.py): batch collation, notes-bus tensors, plan hashing, masks
- [trunk_adapter.py](../../integration/gpt_oss/trunk_adapter.py): trunk loading, dtype/device handling, attention backend selection, gradient checkpointing

## Troubleshooting

**OOM**

- Reduce `batch_size`
- Reduce `max_length`
- Keep `gradient_checkpointing: true`
- Prefer the attention backend your environment actually supports

**Resume after failure**

- Re-run the same command
- The trainer looks for the latest checkpoint in `telemetry_dir` when `resume_from_checkpoint: true`
